# 5.4 SALES TRANSACTION (KASIR)

## 5.4.1 Flow Transaksi Standar

```
1. Kasir login (email/PIN)
2. Pilih shift (buka shift jika belum)
3. Input modal awal (default Rp 200.000)
4. Cari produk:
   ├── Scan barcode (USB scanner)
   ├── Ketik nama/kode produk
   └── Browse kategori
5. Pilih UOM (Sak/Pcs/Dus/Box)
6. Pilih tier harga (Retail/Grosir/Member/dll)
7. Input qty
8. (Opsional) Input customer (untuk piutang/poin/member)
9. (Opsional) Apply diskon (auto dari backoffice atau manual)
10. Review keranjang
11. Pilih payment method (Cash/QRIS/Debit/Kredit/Piutang/Split)
12. Input nominal bayar
13. Cetak struk
14. Transaksi complete
```

## 5.4.2 Fitur Open Bill (Tahan Transaksi)

```
Use case: Customer urgent lain datang, transaksi sekarang perlu ditahan

Flow:
1. Kasir klik "Tahan Transaksi"
2. Input nama/kode untuk identifikasi
3. Transaksi masuk ke list "Open Bills"
4. Lanjutkan transaksi customer lain
5. Lanjutkan open bill:
   ├── Buka list Open Bills
   ├── Pilih yang mau dilanjutkan
   └── Lanjut proses checkout
```

## 5.4.3 Split Payment

```
Customer bayar dengan multiple payment method:
├── Total: Rp 150.000
├── Cash: Rp 50.000
├── QRIS: Rp 100.000
└── Sistem validasi: 50.000 + 100.000 = 150.000 ✅
```

## 5.4.4 Loyalty Points

```
Setup (di Backoffice):
├── Rule: Setiap Rp 10.000 = 1 Point
├── Toggle on/off per cabang
└── Minimum transaksi untuk earn points: (setting)

Penggunaan:
├── Point bisa ditukar di kemudian hari (detail belum defined — OPEN QUESTION)
├── Display point di struk
└── Display total point customer saat transaksi
```

## 5.4.5 Owner Price Override (Tier 7 — Anti-Fraud)

**Konsep:**

Tier harga ke-7 yang **HANYA visible untuk Owner**, untuk prevent fraud dan handle deal khusus.

**Use Case:**
- Owner punya deal khusus dengan customer VIP (harga lebih rendah dari tier manapun)
- Owner tidak ingin kasir tahu harga special ini
- Owner ingin kontrol penuh pricing untuk kasus exceptional

**Flow:**

```
PREREQUISITE:
└── User yang login harus role OWNER (kasir tidak bisa akses)

STEP 1 — Trigger Owner Override:
├── Owner login ke POS (menggunakan akun Owner)
├── Mulai transaksi normal
├── Pilih produk, UOM, qty
├── Klik button "Owner Price Override" (hanya tampil untuk Owner)
└── Sistem minta re-authenticate (PIN Owner)

STEP 2 — Input Manual Price:
├── Sistem tampilkan info:
│   ├── Produk: Pakan Kucing Meow Mix 1kg
│   ├── UOM: Sak
│   ├── Harga Terendah (Tier 1 Retail): Rp 100.000
│   └── "Input harga override:"
├── Owner input manual: Rp 90.000
├── Sistem validasi:
│   ├── Tidak boleh Rp 0
│   ├── Boleh lebih rendah dari harga terendah (no block)
│   └── Warning jika override < 50% dari retail (confirm sekali lagi)
└── Confirm

STEP 3 — Apply Override:
├── Harga di keranjang: Rp 90.000 (bukan tier 1-6)
├── Label di keranjang: "Owner Override" (tidak tampil nominal di kasir)
├── Lanjut checkout normal
└── Print struk: harga Rp 90.000 (no label "override" di struk customer)

STEP 4 — Audit Log:
├── Sistem catat:
│   ├── Transaction ID
│   ├── Product ID & UOM
│   ├── Original price tier (Retail Rp 100.000)
│   ├── Override price (Rp 90.000)
│   ├── Override by (Owner user_id)
│   ├── Timestamp
│   └── Customer ID (jika ada)
└── Log hanya bisa dilihat Owner/Super Admin di Backoffice
```

**Security:**

```
1. HANYA Owner yang bisa akses fitur ini (role-based)
2. Re-authenticate dengan PIN sebelum override (prevent accidental click)
3. Audit trail lengkap (siapa, kapan, produk apa, harga berapa)
4. Kasir/Manager TIDAK BISA lihat harga override di history transaksi
   └── Mereka hanya lihat "Harga Khusus" tanpa nominal
5. Backoffice report: Owner bisa filter transaksi dengan "Owner Override"
```

**Database:**

```sql
-- Tambah field di transaction_items
transaction_items (
  ...existing fields,
  is_owner_override BOOLEAN DEFAULT false,
  override_price DECIMAL(15,2),
  override_by INT REFERENCES users(user_id),
  override_at TIMESTAMP
)

-- Audit log khusus
owner_price_overrides (
  override_id SERIAL PRIMARY KEY,
  transaction_id INT REFERENCES transactions(transaction_id),
  transaction_item_id INT REFERENCES transaction_items(item_id),
  product_id INT,
  uom_id INT,
  original_price DECIMAL(15,2),
  override_price DECIMAL(15,2),
  discount_amount DECIMAL(15,2),
  discount_percentage DECIMAL(5,2),
  override_by INT REFERENCES users(user_id),
  override_at TIMESTAMP DEFAULT NOW(),
  customer_id INT REFERENCES customers(customer_id),
  notes TEXT
)
```
