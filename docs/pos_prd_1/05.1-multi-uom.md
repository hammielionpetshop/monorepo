# 5.1 MULTI-UOM SYSTEM (AUTO-BREAK)

## 5.1.1 Konsep Dasar

Setiap produk bisa memiliki multiple Unit of Measure (UOM):
- **UOM Base** (satuan terkecil): Pcs, Gram, Ml
- **UOM Besar** (satuan kemasan): Sak, Dus, Box, Karton
- **Conversion Ratio**: Variable per produk (1 Sak bisa 20/25/30 pcs tergantung merek)

## 5.1.2 Aturan Stock Multi-UOM

```
Setiap produk memiliki stock TERPISAH per UOM:
├── Stock Sak: 9 (display sebagai "9 Sak")
├── Stock Pcs: 20 (display sebagai "20 Pcs")
└── Total teoritis: (9 × 30) + 20 = 290 pcs (hanya untuk referensi)

❌ Sistem TIDAK convert otomatis saat display
✅ Sistem convert otomatis HANYA saat auto-break saat penjualan
```

## 5.1.3 Algoritma Auto-Break (KRITIKAL — IKUTI PERSIS)

```
FUNCTION sellProduct(productId, uomType, qtyToSell):
  stock = getProductStock(productId)

  # ============================================
  # CASE 1: Jual dalam UOM BESAR (Sak/Dus/Box)
  # ============================================
  IF uomType == stock.uom_besar:
    IF stock.qty_besar >= qtyToSell:
      stock.qty_besar -= qtyToSell
      RETURN success
    ELSE:
      RETURN error("Stock UOM besar tidak cukup")
    # ⚠️ PENTING: Saat jual UOM besar, TIDAK perlu pecah dulu
    # Langsung kurangi stock UOM besar

  # ============================================
  # CASE 2: Jual dalam UOM KECIL (Pcs/Gram/Ml)
  # ============================================
  IF uomType == stock.uom_kecil:

    # Sub-case 2A: Stock kecil cukup
    IF stock.qty_kecil >= qtyToSell:
      stock.qty_kecil -= qtyToSell
      RETURN success

    # Sub-case 2B: Stock kecil TIDAK cukup → AUTO-BREAK
    IF stock.qty_kecil < qtyToSell:
      deficit = qtyToSell - stock.qty_kecil
      sacsNeeded = CEIL(deficit / conversion_ratio)

      # Cek apakah stock UOM besar cukup untuk dipecah
      IF stock.qty_besar < sacsNeeded:
        RETURN error("Total stock tidak cukup")

      # EXECUTE AUTO-BREAK
      stock.qty_besar -= sacsNeeded
      stock.qty_kecil += (sacsNeeded × conversion_ratio)

      # LOG auto-break ke tabel stock_auto_breaks
      LOG auto_break_event(
        product_id, sacsNeeded,
        triggered_by="SALE",
        batch_reference=oldest_batch  # FIFO Strict
      )

      # Kurangi stock kecil sesuai qty
      stock.qty_kecil -= qtyToSell
      RETURN success (with auto_break_flag = true)
```

## 5.1.4 Contoh Skenario

**Skenario 1: Jual dalam UOM Kecil (Cukup)**
```
Stock Awal: 9 Sak, 20 Pcs (1 Sak = 30 Pcs)
Transaksi: Jual 1 Pcs

Process:
├── UOM = Pcs, qty = 1
├── Stock Pcs (20) >= 1 ✅
├── Stock Pcs: 20 - 1 = 19
└── Result: 9 Sak, 19 Pcs
```

**Skenario 2: Jual dalam UOM Kecil (Auto-Break)**
```
Stock Awal: 9 Sak, 20 Pcs (1 Sak = 30 Pcs)
Transaksi: Jual 25 Pcs

Process:
├── UOM = Pcs, qty = 25
├── Stock Pcs (20) < 25 ❌
├── Deficit: 25 - 20 = 5 Pcs
├── Sacs needed: CEIL(5/30) = 1 Sak
├── Stock Sak (9) >= 1 ✅
├── AUTO-BREAK: 1 Sak → +30 Pcs
│   └── Stock: 8 Sak, 50 Pcs
├── Jual 25 Pcs: 50 - 25 = 25 Pcs
└── Result: 8 Sak, 25 Pcs ✅
```

**Skenario 3: Jual dalam UOM Besar**
```
Stock Awal: 9 Sak, 20 Pcs
Transaksi: Jual 2 Sak

Process:
├── UOM = Sak, qty = 2
├── Stock Sak (9) >= 2 ✅
├── Stock Sak: 9 - 2 = 7
└── Result: 7 Sak, 20 Pcs (Pcs tidak berubah)
```

**Skenario 4: Total Stock Tidak Cukup**
```
Stock Awal: 2 Sak, 10 Pcs (1 Sak = 30 Pcs)
Transaksi: Jual 80 Pcs

Process:
├── UOM = Pcs, qty = 80
├── Stock Pcs (10) < 80 ❌
├── Deficit: 80 - 10 = 70 Pcs
├── Sacs needed: CEIL(70/30) = 3 Sak
├── Stock Sak (2) < 3 ❌
└── Result: ERROR "Stock total tidak cukup. Tersedia: 70 Pcs, Diminta: 80 Pcs"
```

## 5.1.5 Manual Break (Hanya untuk Gudang)

```
Manual break HANYA bisa dilakukan oleh Manager/Gudang, bukan kasir.

Use Cases:
├── Kemasan Sak rusak, perlu dipecah jadi Pcs
├── SO menemukan Sak sudah dibuka tapi tidak tercatat
└── Adjustment dari manager

Flow:
1. Manager buka modul "Manual Break Stock"
2. Pilih produk
3. Pilih UOM sumber (Sak) dan qty (1)
4. Input alasan (dropdown: Rusak / SO Adjustment / Lainnya)
5. Butuh PIN approval manager
6. Sistem execute break + log ke stock_auto_breaks dengan triggered_by="MANUAL"
```
