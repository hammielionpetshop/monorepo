# 5.9 DAILY EXPENSES (PENGELUARAN HARIAN)

## 5.9.1 Konsep

Kasir bisa input pengeluaran harian yang dibayar dari uang laci kasir (bukan dari finance kantor).

## 5.9.2 Fitur

```
1. Semua kasir bisa input pengeluaran
2. Tanpa approval (langsung tercatat)
3. Tanpa limit nominal
4. Field input:
   ├── Tanggal (auto: hari ini)
   ├── Kategori (dropdown atau manual)
   ├── Nominal
   ├── Keterangan (wajib)
   └── Upload bukti (optional)
5. Pengeluaran tercatat di shift aktif
6. Muncul otomatis di print settlement
```

## 5.9.3 Kategori Pengeluaran

**Rekomendasi Hybrid (Pre-defined + Manual):**

```
OPSI: Combine pre-defined + manual input
├── Pre-defined (dari backoffice, bisa dikustomisasi):
│   ├── Transport (bensin, parkir, ojek)
│   ├── Konsumsi (makan karyawan)
│   ├── Utilitas (listrik, air, internet)
│   ├── Maintenance (service AC, kebersihan)
│   ├── Supplies (plastik, tisu, ATK)
│   ├── Insentif/Tip karyawan
│   └── Lain-lain
├── Manual: Kasir bisa ketik kategori custom jika tidak ada yang cocok
└── Backoffice review: kategori custom yang sering muncul bisa di-promote jadi pre-defined
```
