# 5.3 FIFO COSTING (STRICT)

## 5.3.1 Konsep FIFO

**FIFO = First In, First Out**

Barang yang masuk **lebih dulu** akan dijual/dihitung sebagai COGS **lebih dulu**.

## 5.3.2 Input Harga Modal dari PO

**Konfirmasi User: OPSI A**

```
Input oleh petugas gudang saat terima barang:
├── Harga per UOM Besar (Sak) = Rp 100.000
└── Sistem AUTO-HITUNG harga per UOM Kecil:
    └── Harga per Pcs = Rp 100.000 / 25 = Rp 4.000

Rumus:
├── Harga Modal UOM Kecil = Harga Modal UOM Besar / Conversion Ratio
└── Harga Modal UOM Lain = Harga Modal UOM Kecil × (Ratio UOM Lain)

Contoh:
Beli: 10 Sak @ Rp 100.000 (1 Sak = 25 Pcs)
├── Harga Modal Sak: Rp 100.000
├── Harga Modal Pcs: Rp 4.000 (auto)
└── Jika ada Dus (1 Dus = 10 Pcs):
    └── Harga Modal Dus: Rp 4.000 × 10 = Rp 40.000 (auto)
```

## 5.3.3 FIFO Strategy (OPSI 1 — Strict)

**Konfirmasi User: OPSI 1**

```
Prinsip: Batch terlama dipecah/dijual dulu, pecahan tetap trace ke batch asal.

Skenario:
Batch 1 (1 Jan):  10 Sak @ Rp 100.000 → Modal Pcs: Rp 4.000
Batch 2 (15 Jan): 10 Sak @ Rp 110.000 → Modal Pcs: Rp 4.400

Transaksi 20 Jan: Jual 30 Pcs

Process:
├── Stock Pcs awal: 0
├── AUTO-BREAK dari Batch 1 (terlama):
│   └── 1 Sak Batch 1 → 25 Pcs Batch 1 @ Rp 4.000
├── Stock setelah break:
│   ├── Batch 1: 9 Sak, 25 Pcs (@ Rp 4.000)
│   └── Batch 2: 10 Sak, 0 Pcs
├── Jual 25 Pcs dari Batch 1 Pcs (habis)
├── Masih butuh 5 Pcs lagi → AUTO-BREAK 1 Sak Batch 1 lagi
│   └── Stock: Batch 1: 8 Sak, 25 Pcs; Batch 2: 10 Sak, 0 Pcs
├── Jual 5 Pcs dari Batch 1
└── Final:
    ├── Batch 1: 8 Sak, 20 Pcs
    ├── Batch 2: 10 Sak, 0 Pcs
    └── COGS = 30 × Rp 4.000 = Rp 120.000
```

## 5.3.4 Batch Tracking (OPSI A — Terpisah)

**Konfirmasi User: OPSI A**

```
Setiap batch tracking per UOM.
Pecahan Sak → Pcs tetap tercatat dari batch mana.

Database structure:
product_stock_batches
├── batch_id
├── product_id
├── branch_id
├── uom_id (sak / pcs)
├── qty
├── cost_price (harga modal)
├── received_date (tanggal terima)
├── po_id (asal PO)
└── parent_batch_id (jika hasil auto-break, reference ke batch asal)

Contoh data:
Batch B001: product_id=1, uom=sak, qty=9, cost=100000, date=1-Jan
Batch B001-pcs: product_id=1, uom=pcs, qty=20, cost=4000,
                date=1-Jan, parent_batch_id=B001
                (ini hasil auto-break dari B001)
```

## 5.3.5 FIFO di Stock Opname

**Konfirmasi User: OPSI A (Dari batch terlama)**

```
Saat SO menemukan selisih MINUS:
├── Asumsi: barang hilang dari batch TERLAMA
├── Kerugian = qty_hilang × harga_modal_batch_terlama
└── Sistem otomatis kurangi qty dari batch terlama

Contoh:
Stock FIFO:
├── Batch 1: 5 Sak @ Rp 100.000 (terlama)
└── Batch 2: 5 Sak @ Rp 110.000

SO menemukan: 9 Sak (harusnya 10)
Selisih: -1 Sak

Asumsi: Hilang dari Batch 1 (terlama)
Kerugian: 1 × Rp 100.000 = Rp 100.000
Stock update: Batch 1 = 4 Sak, Batch 2 = 5 Sak
```
