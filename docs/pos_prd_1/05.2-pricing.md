# 5.2 PRICING SYSTEM (6 TIER + MANUAL OVERRIDE)

## 5.2.1 Struktur Harga

Setiap produk di setiap cabang memiliki **6 tier harga tetap** + **1 tier owner override**:

| Tier | Nama | Use Case | Visibility |
|------|------|----------|------------|
| 1 | Retail | Harga standar customer umum | Semua user |
| 2 | Grosir | Customer beli banyak (tanpa kategori khusus) | Semua user |
| 3 | Member | Customer terdaftar sebagai member | Semua user |
| 4 | Distributor | Reseller/distributor besar | Semua user |
| 5 | Reseller | Reseller kecil | Semua user |
| 6 | Promo | Harga promo khusus | Semua user |
| 7 (Special) | **Owner Price Override** | Manual input harga khusus | **OWNER ONLY** |

## 5.2.2 Aturan Harga

```
1. Harga di-set di Backoffice per produk per cabang
2. Setiap UOM punya 6 harga tier (Tier 1-6)
3. Harga jual per UOM TIDAK HARUS proporsional
   ├── Contoh: Sak Rp 100.000, Pcs boleh Rp 4.500 (bukan 4.000)
   └── Ini normal: jual satuan lebih mahal dari borongan
4. Kasir MEMILIH MANUAL tier harga saat transaksi (Tier 1-6)
   ├── Tidak ada auto-trigger (beli banyak = grosir otomatis) ❌
   └── Kasir yang memutuskan ✅
5. Tier berbeda per cabang (Toko A punya Member, Toko B tidak)
6. TIER 7 — Owner Price Override:
   ├── HANYA OWNER yang bisa akses
   ├── Tidak ada harga pre-set, MURNI input manual nominal
   ├── Butuh re-authenticate (PIN Owner) sebelum override
   ├── Use case: Deal khusus yang hanya owner tahu
   ├── Contoh: Harga terendah (Retail) Rp 100.000, Owner override jadi Rp 90.000
   └── Tercatat di audit log: timestamp, produk, harga original, harga override
```

## 5.2.3 Promo (Override Tier Harga)

```
Backoffice bisa aktivkan "Promo" per cabang:
├── Pilih produk (individual/kategori/brand)
├── Set diskon (%, nominal, BxGy, bundle)
├── Set durasi (start/end date)
├── Set cabang yang berlaku
└── Saat kasir jual produk tersebut di cabang promo:
    └── Sistem auto-apply promo (tidak perlu kasir pilih)
```
