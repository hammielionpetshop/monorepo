# 5.8 DISCOUNT ENGINE

## 5.8.1 4 Tipe Diskon

| Tipe | Deskripsi | Contoh |
|------|-----------|--------|
| Percentage | Diskon persentase | 10%, 20%, 50% |
| Nominal | Potongan rupiah | Rp 5.000, Rp 10.000 |
| Buy X Get Y | Beli X gratis Y | Beli 2 gratis 1 |
| Bundle Pricing | Paket harga khusus | Paket A (3 produk) = Rp 100.000 |

## 5.8.2 Scope Diskon

```
Diskon bisa apply ke:
├── Produk spesifik (by product_id)
├── Kategori produk (by category_id)
├── Brand/Merek (by brand_id)
└── Total transaksi
```

## 5.8.3 Setting dari Backoffice

```
Backoffice set promo:
├── Nama promo
├── Tipe diskon (4 tipe)
├── Value (10%, Rp 5.000, dll)
├── Scope (produk/kategori/brand)
├── Cabang berlaku (bisa multi cabang)
├── Tanggal mulai & berakhir
├── Stackable (default: FALSE)
└── Priority (untuk sorting jika multiple promo eligible)
```

## 5.8.4 Conflict Resolution (Default)

```
DEFAULT: Non-Stackable
├── Jika produk eligible untuk multiple promo
├── Sistem otomatis pilih DISKON TERBESAR
└── Hanya 1 diskon yang apply

Contoh:
Produk X eligible:
├── Diskon Member 10%
├── Diskon Kategori 15%
└── Diskon Promo Produk 20%

Sistem pilih: 20% (terbesar)
```

## 5.8.5 Stackable Override

```
Untuk kasus khusus, backoffice bisa set "Stackable = TRUE" per promo.
Jika stackable aktif dan multiple promo eligible:
├── Semua diskon ter-apply (ditumpuk)
├── Butuh approval manager untuk transaksi dengan stacked discount > threshold
└── Tercatat di audit log

Contoh:
Promo "Lebaran 2026" stackable = TRUE
├── Diskon Member 10% + Diskon Promo 20% = 30% total
└── Hanya 1 promo yang bisa stackable = TRUE pada 1 waktu
```
