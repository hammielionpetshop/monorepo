# 5.7 CUSTOMER DEBT (PIUTANG)

## 5.7.1 Konsep

**Piutang** = Customer berhutang ke toko (belum bayar penuh)

Contoh: Belanja Rp 500.000, bayar Rp 300.000, hutang Rp 200.000.

## 5.7.2 Fitur

```
1. Setiap customer bisa berhutang (tidak ada batasan awal)

2. Limit Piutang Per Customer:
   ├── Toggle on/off per customer
   ├── Jika ON: set max limit (contoh: Rp 1.000.000)
   └── Jika sudah mencapai limit → tidak bisa hutang lagi sampai bayar

3. Cicilan Parsial:
   ├── Customer bisa bayar sebagian (tidak harus lunas)
   └── Sisa hutang tetap tercatat

4. Payment Method saat bayar hutang:
   ├── Cash
   ├── QRIS
   ├── Debit
   └── Transfer

5. History Piutang per Customer:
   ├── Kapan hutang, berapa, kapan bayar, berapa sisa
   └── Tanggal jatuh tempo (optional)
```

## 5.7.3 Flow Bayar Piutang

```
1. Customer datang ke toko mau bayar piutang
2. Kasir buka modul "Piutang Customer"
3. Cari customer
4. Lihat daftar hutang aktif:
   ├── Nota #123 - Rp 200.000 (tgl 10 Apr)
   ├── Nota #456 - Rp 150.000 (tgl 12 Apr)
   └── Total: Rp 350.000
5. Kasir input:
   ├── Nominal bayar: Rp 200.000
   ├── Payment method: Cash
   └── Alokasi: Nota #123 (bayar lunas)
6. Sistem update:
   ├── Hutang Nota #123: Rp 0 (lunas)
   └── Uang masuk ke cash flow
7. Cetak struk pembayaran piutang
```

## 5.7.4 Toggle Setting

```
Backoffice bisa toggle:
├── "Aktifkan Fitur Piutang" (global on/off)
├── "Izinkan Piutang Customer Non-Member" (on/off)
└── "Max Default Limit Piutang" (per customer)
```
