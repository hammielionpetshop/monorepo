## 🗄️ DB Strategy (Penting — Baca Dulu)

Karena DB sudah punya tabel lama dari `hml-desktop`, kita **tidak akan menyentuh tabel yang sudah ada** di schema `public`.

**Solusi: Gunakan PostgreSQL Schema Namespace `petshop`**

```sql
-- Semua tabel baru dibuat di dalam schema "petshop"
-- Tabel lama di schema "public" tidak disentuh sama sekali

CREATE SCHEMA IF NOT EXISTS petshop;

-- Contoh:
CREATE TABLE petshop.users (...);       -- ← tabel baru kita
-- vs
CREATE TABLE public.user (...);         -- ← tabel lama (tidak diubah)
```

Ini berarti:
- ✅ Tidak ada risiko data lama terhapus
- ✅ Bisa rollback dengan `DROP SCHEMA petshop CASCADE`
- ✅ Bisa migrate data lama ke schema baru sesuai kebutuhan nanti
- ✅ Drizzle hanya manage schema `petshop` saja

**API Routes Phase 1:**

| Method | Path | Logic |
|--------|------|-------|
| `GET` | `/api/pos/health` | return `{ status: 'ok', timestamp }` |
| `POST` | `/api/pos/auth/login` | mode: `staff_pin` atau `email_password` |
| `POST` | `/api/pos/auth/refresh` | verify refresh token → new access token |
| `POST` | `/api/pos/auth/logout` | (stateless, client-side clear) |

**Login logic detail:**

```
mode = staff_pin:
1. Cari user WHERE staff_number = input AND is_active = true
2. Verify PIN (argon2.verify(user.pin_hash, input.pin))
3. Return JWT dengan permissions[] dari role

mode = email_password:
1. Cari user WHERE email = input AND is_active = true
2. Verify password (argon2.verify(user.password_hash, input.password))
3. Return JWT dengan permissions[] dari role
```

**`.env.local` yang dibutuhkan:**
