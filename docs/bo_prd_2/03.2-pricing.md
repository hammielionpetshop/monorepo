# 3.2 MULTI-TIER PRICING (7 TIERS)

## 3.2.1 Pricing Overview

**Setiap produk di setiap cabang punya:**
- **6 tier harga tetap** (Retail, Grosir, Member, Distributor, Reseller, Promo) — di-set di Backoffice
- **1 tier owner override** (manual input di POS, owner-only) — lihat docs/pos_prd_1/05.4-sales-transaction.md

**PENTING:** Tier 7 (Owner Override) **TIDAK di-set di Backoffice**, murni manual input di POS saat transaksi.

## 3.2.2 Pricing Structure per UOM

```
Contoh Produk: Pakan Kucing Meow Mix

UOM 1: Sak (30 Pcs)
├── Tier 1 (Retail):      Rp 100.000
├── Tier 2 (Grosir):      Rp  95.000
├── Tier 3 (Member):      Rp  90.000
├── Tier 4 (Distributor): Rp  85.000
├── Tier 5 (Reseller):    Rp  88.000
└── Tier 6 (Promo):       Rp  80.000

UOM 2: Pcs
├── Tier 1 (Retail):      Rp 4.500 ← TIDAK harus proporsional!
├── Tier 2 (Grosir):      Rp 4.200
├── Tier 3 (Member):      Rp 4.000
├── Tier 4 (Distributor): Rp 3.800
├── Tier 5 (Reseller):    Rp 3.900
└── Tier 6 (Promo):       Rp 3.500

Notes:
- Harga Pcs BUKAN (Harga Sak ÷ 30)
- Jual eceran lebih mahal = normal business practice
- Owner set manual per UOM
```

## 3.2.3 Pricing per Cabang

**Harga bisa BERBEDA per cabang:**

```
Produk: Pakan Kucing Meow Mix (UOM: Sak)

Cabang Sudirman:   Retail Rp 100.000 | Grosir Rp 95.000
Cabang Kelapa:     Retail Rp 105.000 | Grosir Rp 100.000 ← area premium
Cabang Gatsu:      Retail Rp  98.000 | Grosir Rp  93.000 ← kompetisi ketat
```

**Database Design:**
```sql
product_pricing (
  pricing_id SERIAL PRIMARY KEY,
  product_id INT REFERENCES products(product_id),
  branch_id INT REFERENCES branches(branch_id),
  uom_id INT REFERENCES product_uoms(id),
  tier_1_retail DECIMAL(15,2),
  tier_2_grosir DECIMAL(15,2),
  tier_3_member DECIMAL(15,2),
  tier_4_distributor DECIMAL(15,2),
  tier_5_reseller DECIMAL(15,2),
  tier_6_promo DECIMAL(15,2),
  -- Tier 7 (Owner Override) TIDAK ada di sini
  minimum_price DECIMAL(15,2),
  is_active BOOLEAN DEFAULT true,
  effective_date DATE,
  created_at TIMESTAMP DEFAULT NOW(),
  updated_at TIMESTAMP DEFAULT NOW(),
  UNIQUE(product_id, branch_id, uom_id)
)
```

## 3.2.4 Bulk Update Harga

### Fitur 1: Bulk Update by Filter

```
FLOW:
1. Owner buka "Bulk Update Harga"
2. Filter produk (by kategori, brand, supplier, tag)
3. Pilih action:
   ├── Increase All by %: +10%
   ├── Decrease All by %: -5%
   ├── Increase All by Amount: +Rp 5.000
   └── Decrease All by Amount: -Rp 2.000
4. Preview: "100 produk akan berubah harga. Total delta: +Rp 10 juta"
5. Confirm → Harga updated
6. Audit log: "User X bulk updated 100 products (+10%)"
```

### Fitur 2: Copy Harga Antar Cabang

```
Use case: Cabang baru, copy harga dari cabang existing

FLOW:
1. Owner buka "Copy Harga Antar Cabang"
2. Pilih source cabang & target cabang
3. Option: Copy SEMUA / by kategori / by brand
4. Adjustment (optional): No adjustment / +% / -%
5. Preview: "500 produk akan di-copy dengan harga +5%"
6. Confirm → Harga copied
7. Audit log: "User X copied prices from Sudirman to XYZ (+5%)"
```

### Fitur 3: Excel Import/Export

```
EXPORT:
1. Owner klik "Export Harga ke Excel"
2. Filter: Cabang mana? Kategori apa?
3. Download file: "harga_cabang_sudirman_2026-04-17.xlsx"

File structure:
┌─────────────────────────────────────────────────────────┐
│ SKU     | Nama       | UOM | Retail | Grosir | Member │
├─────────────────────────────────────────────────────────┤
│ PKM-001 | Pakan Meow | Sak | 100000 | 95000  | 90000  │
│ PKM-001 | Pakan Meow | Pcs | 4500   | 4200   | 4000   │
└─────────────────────────────────────────────────────────┘

IMPORT:
1. Download template Excel
2. Fill data
3. Upload file
4. Sistem validasi (SKU exist, UOM exist, harga > 0, format benar)
5. Preview changes (highlight yang berubah)
6. Confirm → Harga updated
7. Audit log: "User X imported 500 price changes via Excel"
```

## 3.2.5 Harga Minimum Warning

```
Per produk, Owner bisa set "Harga Minimum" (optional):
├── Use case: Prevent selling below cost
├── Warning only (tidak block transaksi)
└── Audit log: Track semua transaksi di bawah harga minimum

Jika kasir pilih harga < harga minimum:
└── Warning pop-up: "Harga di bawah minimum! Yakin?"
    ├── Kasir bisa lanjut (tidak di-block)
    └── Tercatat di audit log
```

## 3.2.6 Scheduled Price Change

```
Use case: Owner mau ganti harga efektif mulai tanggal X

FLOW:
1. Owner edit harga produk
2. Centang "Schedule Price Change"
3. Input:
   ├── Effective Date: 18 April 2026, 00:00
   └── New Price: Rp 110.000 (dari Rp 100.000)
4. Save → Status: SCHEDULED
5. Sistem auto-apply harga baru saat tanggal tiba
6. Notifikasi ke Manager: "Harga 50 produk berubah hari ini"
```
