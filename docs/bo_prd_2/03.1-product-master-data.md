# 3.1 PRODUCT MASTER DATA

## 3.1.1 Product Fields (Complete)

```
PRODUCT MASTER:
├── SKU (required, unique, alphanumeric)
│   └── Format: Auto-generate atau manual input
│   └── Contoh: "PKM-001", "OBT-KUC-123"
│
├── Barcode (optional, bisa multiple)
│   ├── Primary barcode (EAN-13, UPC, dll)
│   └── Additional barcodes (array)
│   └── Use case: 1 produk bisa punya barcode kemasan besar & kecil
│
├── Nama Produk (required, max 200 characters)
│
├── Deskripsi (optional, rich text)
│
├── Kategori (required, dropdown, 3-level hierarchy)
│
├── Brand (optional, dropdown)
│
├── Supplier (multiple, multi-select)
│   └── 1 produk bisa dari >1 supplier
│
├── Berat (optional)
│   ├── Value (decimal)
│   └── Unit (gram/kg)
│
├── Expired Tracking (boolean)
│   ├── TRUE: Produk punya tanggal expired (obat, makanan)
│   ├── FALSE: Produk tidak punya expired (aksesoris, mainan)
│   └── Jika TRUE → di PO/receiving wajib input tanggal expired
│
├── Tags/Labels (array)
│   ├── Auto-tag by sistem: "Best Seller", "New Arrival", dll
│   └── Manual tag by admin: "Promo Lebaran", "Recommended"
│
├── Status (dropdown)
│   ├── Active (dijual)
│   ├── Inactive (tidak dijual, tapi masih ada stock)
│   └── Discontinued (tidak dijual lagi, stock habis)
│
├── Foto Produk (multiple images)
│   ├── Primary image (required)
│   ├── Additional images (optional, max 5)
│   └── Upload & store di cloud storage
│
├── Created By (auto, user_id)
├── Created At (auto, timestamp)
├── Updated By (auto, user_id)
└── Updated At (auto, timestamp)
```

## 3.1.2 Product CRUD Operations

### CREATE Product

```
FLOW:
1. Manager Backoffice/Owner klik "Tambah Produk"
2. Form input (5 tabs):
   ├── Tab 1: Basic Info (SKU, nama, kategori, brand, deskripsi)
   ├── Tab 2: UOM & Pricing (lihat 03.2-pricing.md)
   ├── Tab 3: Supplier & Stock (supplier, berat, expired tracking)
   ├── Tab 4: Images (upload foto)
   └── Tab 5: Tags & Status
3. Validasi:
   ├── SKU unique check
   ├── Barcode unique check (jika ada)
   ├── Kategori exist
   └── Minimal 1 UOM dengan harga
4. Save → Product created
5. Audit log: "User X created product Y at timestamp Z"
```

**Filter Options:**
- By kategori, brand, status, tag, supplier
- By expired tracking (Yes/No)
- By stock status (In Stock / Low Stock / Out of Stock)

**Search:** By SKU (partial), nama produk (partial), barcode (exact)

### UPDATE Product

```
FLOW:
1. Klik "Edit" pada produk
2. Form sama seperti Create (pre-filled)
3. Edit field yang mau diubah
4. Save → Product updated
5. Audit log:
   "User X updated product Y:
    - Nama: 'Pakan Meow' → 'Pakan Meow Mix'
    - Harga Retail Sak: Rp 100.000 → Rp 110.000"
```

**Business Rule — Harga Edit Approval:**

```
IF harga berubah > 10% dari harga sebelumnya:
└── Butuh approval Owner
    ├── Submit → Status: PENDING_APPROVAL
    ├── Owner approve → Harga updated
    └── Owner reject → Harga tidak berubah
```

### DELETE Product

```
FLOW:
1. Klik "Delete" pada produk
2. Sistem cek:
   ├── Stock > 0 di semua cabang? → Block
   ├── Pernah ada di transaksi? → Block
   └── Jika OK:
       ├── Confirmation dialog
       ├── Butuh approval Owner (always)
       └── Owner approve → Soft delete
3. Audit log: "User X deleted product Y (SKU: Z)"
```

### COPY Product (Duplicate)

```
Use case: Buat produk mirip dengan produk existing

FLOW:
1. Klik "Copy" pada produk
2. Sistem duplicate semua field KECUALI:
   ├── SKU (auto-generate baru atau input manual)
   ├── Barcode (kosong, harus input baru)
   └── Status (default: Inactive)
3. User edit field yang perlu diubah
4. Save → New product created
```
