# 3.6 UOM (UNITS OF MEASURE)

## 3.6.1 UOM Structure

**Linked to:** docs/pos_prd_1/05.1-multi-uom.md

```
Per produk bisa punya multiple UOM dengan conversion ratio.

Contoh: Pakan Kucing Meow Mix
├── UOM Base (kecil): Pcs
└── UOM Besar: Sak
    └── Conversion: 1 Sak = 30 Pcs (variable per produk)
```

**Database:**
```sql
product_uoms (
  id SERIAL PRIMARY KEY,
  product_id INT REFERENCES products(product_id),
  uom_name VARCHAR(50), -- "Sak", "Pcs", "Box", "Dus"
  is_base BOOLEAN, -- TRUE untuk UOM terkecil
  conversion_to_base DECIMAL(10,2), -- 1 Sak = 30 Pcs
  created_at TIMESTAMP
)
```

## 3.6.2 UOM Setting di Backoffice

```
EDIT PRODUCT → Tab UOM:
┌────────────────────────────────────────┐
│ UOM Kecil (Base): Pcs  [✓]            │
│                                        │
│ UOM Besar:                             │
│ ├── Sak   [Conversion: 30 Pcs]        │
│ └── [+ Tambah UOM]                     │
└────────────────────────────────────────┘
```

## 3.6.3 Business Rules

- Minimal 1 UOM per produk (UOM base)
- Conversion ratio bisa berbeda per produk (1 Sak bisa 20, 25, atau 30 Pcs)
- Harga per UOM tidak harus proporsional (lihat 03.2-pricing.md section 3.2.2)
- Auto-break logic saat penjualan dihandle di POS (lihat docs/pos_prd_1/05.1-multi-uom.md)
