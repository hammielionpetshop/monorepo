# 3.3 CATEGORIES MANAGEMENT

## 3.3.1 Category Hierarchy (3-Level)

```
STRUCTURE:
Level 1 (Main Category)
├── Level 2 (Sub Category)
│   └── Level 3 (Sub-Sub Category)

CONTOH:
Makanan
├── Makanan Kucing
│   ├── Dry Food Kucing
│   └── Wet Food Kucing
├── Makanan Anjing
│   ├── Dry Food Anjing
│   └── Wet Food Anjing
└── Makanan Burung

Obat
├── Obat Kucing
│   ├── Vitamin Kucing
│   └── Anti-parasit Kucing
├── Obat Anjing
└── Obat Burung

Aksesoris
├── Aksesoris Kucing
│   ├── Kalung & Tali
│   └── Mainan Kucing
├── Aksesoris Anjing
└── Aksesoris Burung
```

**Interpretasi requirement user ("Cat > dry food, wet food, obat, acc"):**
- Level 1: Makanan, Obat, Aksesoris (main category)
- Level 2: By pet type (Kucing, Anjing, Burung)
- Level 3: Specific type (Dry Food, Wet Food, Vitamin, dll)

## 3.3.2 Category CRUD

**Fields:**
```
CATEGORY:
├── Category ID (auto, PK)
├── Category Name (required)
├── Parent Category ID (nullable, FK to categories)
│   └── NULL = Level 1 (root)
│   └── NOT NULL = Level 2/3 (child)
├── Level (auto-calculated: 1, 2, or 3)
├── Description (optional)
├── Icon/Image (optional, for UI)
├── Sort Order (for display order)
├── Status (Active/Inactive)
├── Created At
└── Updated At
```

**CREATE:**
```
1. Klik "Tambah Kategori"
2. Input: nama, parent category (optional), deskripsi, icon
   └── Jika parent kosong → Level 1
   └── Jika ada parent → Level 2/3
3. Save → Category created
```

**READ (Tree View):**
```
📁 Makanan (50 produk)
  📁 Makanan Kucing (30 produk)
    📄 Dry Food Kucing (15 produk)
    📄 Wet Food Kucing (15 produk)
  📁 Makanan Anjing (20 produk)
    📄 Dry Food Anjing (10 produk)
    📄 Wet Food Anjing (10 produk)
```

**UPDATE:**
```
1. Klik "Edit" pada kategori
2. Edit nama, parent, deskripsi
3. Save → Updated
```

**DELETE:**
```
Validasi:
├── Ada produk di kategori ini? → Block
├── Ada child category? → Block (hapus child dulu)
└── Jika OK → Soft delete
```
