# 5.2 USER & RBAC MANAGEMENT

## 5.2.1 User Management (CRUD)

**Target User:** Super Admin, Owner

**User Fields:**
```
CREATE USER:
├── Nama Lengkap (required)
├── Email (required, unique) → untuk login
├── PIN (4-6 digit, required) → untuk POS quick login
├── Phone (optional)
├── Role (dropdown, required):
│   ├── Owner
│   ├── Super Admin
│   ├── Finance
│   ├── Manager Backoffice
│   ├── Manager Toko
│   ├── Kasir
│   └── Petugas Gudang
├── Cabang Assignment (multi-select):
│   └── 1 user bisa assign ke multiple cabang
├── Foto Profil (optional, upload)
├── Status: Active / Inactive
└── Join Date (auto)
```

**Business Rules:**
- Email must be unique (validation)
- PIN must be 4-6 digit numeric
- Role "Owner" hanya bisa dibuat oleh Super Admin
- Tidak ada limit jumlah user per role/cabang
- 1 user bisa handle multiple cabang (contoh: Manager A handle 2 toko)

**UI Flow:**
```
USER LIST PAGE:
┌────────────────────────────────────────────────┐
│ [+ Tambah User]        [Search: ____] [Filter]│
├────────────────────────────────────────────────┤
│ Nama      | Email        | Role    | Cabang   │
├────────────────────────────────────────────────┤
│ Ahmad     | ahmad@...    | Kasir   | Sudirman │ [Edit] [Delete]
│ Budi      | budi@...     | Manager | Kelapa   │ [Edit] [Delete]
│ ...       |              |         |          │
└────────────────────────────────────────────────┘
```

## 5.2.2 Role-Based Access Control (RBAC)

**Pre-defined Roles:**

| Role | Description | Permissions |
|------|-------------|-------------|
| **Owner** | Business owner | Full access (God mode) |
| **Super Admin** | IT admin | User management, settings, audit log (no finance) |
| **Finance** | Financial controller | Finance module, approve void/PO, view reports |
| **Manager Backoffice** | Operations manager | Master data CRUD, approve SO, view inventory |
| **Manager Toko** | Store manager | View cabangnya saja, approve minor requests |
| **Kasir** | Cashier (POS only) | POS access only (no backoffice) |
| **Petugas Gudang** | Warehouse staff | PO receiving, stock input |

**Permission Matrix:**

| Module/Feature | Owner | Super Admin | Finance | Manager BO | Manager Toko | Kasir | Gudang |
|----------------|-------|-------------|---------|------------|--------------|-------|--------|
| **Dashboard** | | | | | | | |

| **Master Data** | | | | | | | |

## 5.2.3 Customizable Permissions

**Owner/Super Admin bisa customize permission per user:**

```
EDIT USER PERMISSION:
User: Ahmad (Role: Manager Backoffice)

┌────────────────────────────────────┐
│ ☑ Dashboard - View All Cabang     │
│ ☑ Products - CRUD                  │
│ ☐ Products - Delete (override)    │ ← Custom toggle
│ ☑ Inventory - View Stock           │
│ ☐ Finance - View P&L (override)   │ ← Custom toggle
│ ☑ Approval - Void                  │
│ ☑ Approval - SO                    │
│ ☐ Approval - PO >5jt (override)   │ ← Custom toggle
└────────────────────────────────────┘
[Save Changes]
```

**Inheritance:**
- Start from role default permissions
- Owner toggle on/off specific permissions
- Changes saved per user (override role default)

## 5.2.4 Audit Log

**What to Log:**

| Event | Log Details |
|-------|-------------|
| Login/Logout | User, timestamp, IP address |
| CRUD Master Data | User, action, entity, before/after value |
| Approval Actions | User, action (approve/reject), entity, request_id, timestamp |
| Stock Adjustment | User, product, qty change, reason, timestamp |
| Financial Transaction | User, type, amount, timestamp |
| Setting Changes | User, setting changed, old value, new value, timestamp |
| Permission Changes | Admin user, target user, permission changed, timestamp |

**Audit Log Table:**
```
AUDIT LOG:
┌────────────────────────────────────────────────────────────────┐
│ Timestamp  | User    | Action        | Entity     | Details   │
├────────────────────────────────────────────────────────────────┤
│ 17/4 14:30 | Owner   | Approve       | PO-123     | Rp 10jt   │
│ 17/4 14:25 | Ahmad   | Update        | Product-45 | 100k→110k │
│ 17/4 14:20 | Budi    | Login         | -          | IP: x.x.x │
└────────────────────────────────────────────────────────────────┘

[Filter: User | Action | Date Range] [Export CSV]
```

**Retention:** 5 tahun

**Access:** Owner & Super Admin only

## 5.2.5 Authentication

**Login Methods:**
1. **Email + Password** (primary)
2. **PIN** (untuk POS quick login)

**Session Management:**
- JWT token di HTTP-only cookie
- Auto-logout setelah 30 menit idle (configurable)
- Remember me (optional checkbox)

**Security:**
- Password hashed (bcrypt/argon2)
- PIN hashed juga
- No 2FA (sesuai requirement user: "tidak perlu otp")
- No IP whitelist (akses dari mana saja)
